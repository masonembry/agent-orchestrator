# UI Specification — Expert Workspace Offer Flow

Source of truth for `expert-workspace` UI behavior in the sales offer flow.
Format: [OpenSpec](https://github.com/Fission-AI/OpenSpec)

This spec reflects the system as of the Dynamic Smart Offer (RDESQ-1900) implementation.

---

## Purpose

The offer flow presents a personalized, AI-assisted checklist to the expert during a customer call. It displays tone guidance and pitch points generated by Logic Builder from real-time transcript signals, with static fallbacks when dynamic content is unavailable.

---

## Requirements

### Requirement: Feature flag gating

The Dynamic Smart Offer feature MUST be gated by the `isDynamicSmartOfferEnabled` feature flag. When the flag is off, the offer flow renders in standard (non-dynamic) mode with no behavioral change.

#### Scenario: Flag is off

- GIVEN the `isDynamicSmartOfferEnabled` feature flag is `false`
- WHEN an expert opens the offer flow
- THEN the standard `ChecklistSection` renders for all sections
- AND no dynamic components are mounted
- AND no WebSocket listener for Logic Builder events is registered

#### Scenario: Flag is on

- GIVEN the `isDynamicSmartOfferEnabled` feature flag is `true`
- AND the sales checklist has a section with `sectionType === "coreBenefits"` and `enableDynamicContent === true`
- WHEN an expert opens the offer flow
- THEN `isEnabled` is `true` and dynamic rendering activates

---

### Requirement: Tone Guidance section

`DynamicToneGuidance` MUST render above the offer accordion when the feature is enabled, showing one of three states: loading, awaiting data, or loaded content.

#### Scenario: Loading state (2s pending delay)

- GIVEN the feature flag is on and the offer flow is open
- WHEN the 2-second `isPending` delay is active (on mount or after "Show latest")
- THEN `DynamicToneGuidance` renders a skeleton placeholder

#### Scenario: Loaded state — valid data

- GIVEN the `isPending` delay has completed
- AND `data.isValid()` returns `true`
- WHEN the expert views the offer flow
- THEN `DynamicToneGuidance` renders the AI-generated `toneGuidance` string

#### Scenario: Awaiting state — no valid data yet

- GIVEN the `isPending` delay has completed
- AND `data.isValid()` returns `false`
- WHEN the expert views the offer flow
- THEN `DynamicToneGuidance` renders the awaiting UI: animated Lottie spark icon, bold "Listening for customer cues" title, and body copy in a neutral card

#### Scenario: No dismiss button

- GIVEN `DynamicToneGuidance` is visible in any state
- WHEN the expert views the offer flow
- THEN there is no close or dismiss button — the section is always visible while the feature is enabled

---

### Requirement: Dynamic Pitch Points (Core Benefits slot)

`DynamicCoreBenefits` MUST replace the static Core Benefits section when the feature is enabled, or fall back to the static section when dynamic data is unavailable.

#### Scenario: Dynamic data available

- GIVEN `isEnabled` is `true`
- AND `data.isValid()` returns `true`
- AND the `isPending` delay is not active
- WHEN the expert views the offer flow
- THEN `DynamicCoreBenefits` renders the AI-generated pitch points in place of the static Core Benefits section

#### Scenario: Awaiting data

- GIVEN `isEnabled` is `true`
- AND `data.isValid()` returns `false`
- AND the `isPending` delay is not active
- WHEN the expert views the offer flow
- THEN `DynamicCoreBenefits` renders the static Core Benefits `ChecklistSection` as a fallback
- AND the static and dynamic sections are never shown simultaneously

#### Scenario: Loading state

- GIVEN the `isPending` delay is active
- WHEN the expert views the offer flow
- THEN `DynamicCoreBenefits` renders a skeleton with a Sparkle icon accordion header

#### Scenario: Feature disabled

- GIVEN `isEnabled` is `false`
- WHEN the expert views the offer flow
- THEN the standard `ChecklistSection` renders for Core Benefits with no dynamic behavior

---

### Requirement: "Show latest" button

A "Show latest" button MUST appear when newer Logic Builder data has arrived while the expert is viewing loaded content, and MUST NOT appear otherwise.

#### Scenario: No newer data

- GIVEN valid data is currently displayed
- AND no new `dynamic_smart_offer` event has arrived since the current data was shown
- WHEN the expert views the offer flow
- THEN no "Show latest" button is rendered

#### Scenario: Newer data arrives while viewing loaded content

- GIVEN valid data is currently displayed
- WHEN a new `dynamic_smart_offer` WebSocket event is received
- THEN `hasNewerData` becomes `true`
- AND "Show latest" renders alongside the current content

#### Scenario: Expert clicks "Show latest"

- GIVEN `hasNewerData` is `true` and "Show latest" is visible
- WHEN the expert clicks "Show latest"
- THEN a 2-second `isPending` delay begins
- AND after the delay, the latest store data is displayed

---

### Requirement: Auto-refresh on new data during awaiting state

When the expert is in the awaiting state and new Logic Builder data arrives, the UI MUST automatically trigger the pending delay and display the new content without any user action.

#### Scenario: New data arrives during awaiting state

- GIVEN `isAwaitingData` is `true` (feature on, data not yet valid)
- WHEN a new `dynamic_smart_offer` WebSocket event is received with valid data
- THEN `useDynamicSmartOffer` automatically triggers the 2-second `isPending` delay
- AND after the delay, the latest store data is displayed with no user action required

---

### Requirement: "Example of Coverage" section hidden

The "Example of Coverage" / "Personalize the Coverage" (`CoverageCarousel`) section MUST NOT render when the Dynamic Smart Offer feature is enabled.

#### Scenario: Feature enabled — all states

- GIVEN `isEnabled` is `true`
- WHEN the expert is in the offer flow (any state: pending, awaiting, or loaded)
- THEN the `CoverageCarousel` section is not rendered

---

### Requirement: AI content indicator

A Sparkle (✦) icon MUST appear as a consistent visual indicator on all AI-generated content within the offer flow.

#### Scenario: Tone Guidance and Pitch Points accordion header

- GIVEN AI-generated content is displayed
- WHEN the expert views the offer flow
- THEN a Sparkle icon appears on the Tone Guidance section and the Pitch Points accordion header
